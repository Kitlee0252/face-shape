# TryOn AI — 虚拟试戴功能规划

## 状态：待开发（后期功能）

## 一句话描述

用户上传照片检测脸型后，点击 "Try it on me"，AI 实时生成用户本人戴上推荐发型/眼镜的效果图。

## 核心价值

- 当前：所有同脸型用户看到相同的静态推荐图
- TryOn AI：生成**用户本人**的试戴效果图，从"你适合 bob cut"升级为"你戴 bob cut 长这样"
- 全网同类工具（faceshapedetector.ai 等）均无此功能，属于品类级差异化

## 用户流程

```
上传照片
  → 浏览器端 MediaPipe 检测脸型（现有流程，不变）
  → 展示推荐列表 + 静态参考图（现有流程，不变）
  → 用户点击 "Try it on me" 按钮
  → 弹窗告知：照片将发送到 AI 服务处理，用户明确同意
  → 照片 + prompt 发送到 Flux Kontext API
  → AI 生成用户 + 该发型/眼镜的合成图（~12 秒）
  → 返回展示效果图
```

## 隐私策略

- 基础功能（检测 + 静态推荐）：仍然 100% 浏览器端，零上传
- TryOn AI：用户主动触发 + 明确同意后才上传处理
- 比竞品"强制上传才能看结果"更诚实

## 技术方案

- **API**：Kie.ai Flux Kontext（已有密钥和生成脚本）
- **生成速度**：~12 秒/张
- **需要新增**：
  - 后端 API 路由（代理 Kie.ai 调用，保护 API key）
  - 前端 TryOn 按钮 + 同意弹窗 + loading 状态 + 结果展示
  - 用量计数 + 免费额度限制逻辑
- **部署调整**：当前 `output: 'export'` 纯静态，需要改为支持 API routes（Cloudflare Workers / Pages Functions）
